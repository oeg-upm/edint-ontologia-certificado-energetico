Certificado Energético: Descripción volver a índice

En la Figura 1 se muestra el modelo conceptual de la ontología con sus clases y relaciones (propiedades de objeto).

La ontología modela los certificados de eficiencia energética de edificios según el Real Decreto 390/2021. Cada certificado tiene una calificación energética (con indicadores de energía primaria, emisiones CO₂, demandas y consumos por uso), una etiqueta energética (letra A–G) y un procedimiento de cálculo (CEP, CEH, CE3X, etc.).

El certificado es emitido por un técnico competente y puede ir dirigido al propietario o inquilino mediante las recomendaciones de uso. Si el edificio tiene margen de mejora, el certificado incluye recomendaciones de mejora (sobre envolvente, instalaciones, automatización o combinadas), con ahorro estimado, coste, periodo de retorno y la letra mejorada resultante.

Las clasificaciones (tipo de certificado, uso del edificio, zona climática, fuente de energía, tipo de procedimiento, letra de calificación, versión de escala, tipo de recomendación, destinatario, base de datos, vía de acceso y módulo formativo) se representan mediante propiedades de objeto que apuntan a conceptos SKOS, organizados en esquemas de conceptos independientes disponibles en la carpeta kos/.

El certificado puede tener un certificado predecesor (renovación o cambio de fase: proyecto → obra terminada) y, en caso de obra terminada, se indica si se ha utilizado la misma versión del procedimiento de cálculo (Art. 9.4 RD 390/2021).
En el ejemplo anterior se muestra una porción del grafo de conocimiento en RDF para un certificado de eficiencia energética de un edificio existente. El certificado (tipo ExistingBuilding según SKOS) es emitido por un técnico competente y contiene una calificación energética (letra D, versión de escala RD 390/2021, base calculada) y una etiqueta energética (letra D). Los datos del edificio incluyen uso residencial, zona climática D3 y superficie útil. Las clasificaciones (tipo de certificado, uso, zona climática, letra, tipo de procedimiento, base de datos, tipo de recomendación, destinatario) se representan mediante conceptos SKOS de los esquemas disponibles en kos/. La recomendación de mejora (tipo instalación) estima una mejora a letra B con un ahorro del 38 %.
