Certificado de Eficiencia Energética: Descripción volver a índice

En la Figura 1 se muestra el modelo conceptual de la ontología con sus clases y relaciones (propiedades de objeto).

La ontología modela el certificado de eficiencia energética como la unidad documental central, según el Real Decreto 390/2021. El certificado está vinculado a una calificación energética (que incluye la letra de calificación global en la escala A-G y los indicadores numéricos de consumo de energía primaria y emisiones de CO₂), una etiqueta energética (la representación visual), un técnico competente (que firma el certificado), una o más recomendaciones de mejora (intervenciones técnicas para el propietario del edificio), y recomendaciones de uso (recomendaciones de comportamiento u operativas para el ocupante del edificio).

El procedimiento de cálculo registra si la calificación se obtuvo mediante el procedimiento general o simplificado, y si los datos son calculados, medidos o mixtos. Las características del edificio (uso, zona climática, fuentes de energía para cada sistema técnico) se registran como propiedades de dato o de objeto que enlazan con esquemas de conceptos SKOS.

Los vocabularios controlados (tipo de certificado, uso de edificio, zona climática, fuente de energía, letra de calificación, versión de escala, tipo de recomendación, tipo de procedimiento, base de datos, tipo de destinatario, vía de acceso y módulo del curso) se representan mediante propiedades de objeto que apuntan a conceptos SKOS, organizados en esquemas de conceptos independientes disponibles en la carpeta kos/.

El edificio se modela como una entidad externa que referencia al certificado mediante una propiedad definida en su propia ontología (e.g., BOT, schema.org). Esta separación permite que la ontología del certificado se mantenga centrada en la unidad documental mientras permite la integración con bases de datos de edificios.
